## Provider Console Updates

### UI Alignment with Deploy Console

Deval shared an update on current development work:

* Provider Console UI is being redesigned to **match the Deploy Console interface**.
* The goal is to create a **consistent experience across provider and deploy tools**.
* The updated interface will be released soon.

### Development Transparency

* All development progress can be tracked in the Console repository.
* Community members can view milestones, issues, and development progress.
* Deval and the team remain active in Discord assisting providers with console issues.

## Network Upgrades Impacting Providers

Tyler highlighted upcoming network upgrades that providers should prepare for.

### Burn Mint Equilibrium (BME) Testnet

* The BME incentivized testnet is currently active.
* Testing is **CLI-based** and involves real providers on the network.
* This testnet supports the broader BME economic mechanism rollout.

### Smart Contracts Integration

* Smart contracts will be introduced as part of the BME rollout.
* This will enable oracle functionality required for the BME system.

### Lease Closed Issue Fix

* A network upgrade will address an issue affecting the **lease_closed_reason** field.

* The current issue has caused friction for users interacting with the marketplace.
* The upgrade will ensure that lease closures properly record the reason on-chain.

---

## Provider Infrastructure Improvements

### Log Forwarding & Event Forwarding

* Core team is implementing **log forwarding** capabilities.
* This feature will allow better debugging and monitoring of deployments.
* It has been requested for a long time by the Console team and providers.

### Current Status

* Code currently in testing by core team and client teams.
* Upgrade instructions will be announced in provider channels.

---

## Provider Audit Improvements

Tyler announced upcoming improvements to the **provider audit system**.

### Background

* Historically audits were handled by third-party services such as M3ter/Moltry audits.
* Some external audit services were discontinued due to business sustainability challenges.

### Current Situation

* The core team currently provides **free provider audits**.

### Future Direction

The Akash team is exploring a **new provider audit framework**, including:

* Improved trust verification mechanisms
* Trusted execution and compute validation
* Better monitoring and verification of providers

### Upcoming Deliverables

* A formal **audit specification document** will be written and shared with the community.

### Pricing Discussion

* Community members asked whether audits may become paid services.
* Tyler clarified the intent is **not to monetize audits** but to maintain:

  * a free or minimal-cost audit system
  * an open-source trust layer for the marketplace.

---

## Community Questions

### Audit Pricing

Rodri asked whether the future audit framework might include paid tiers similar to earlier services.

Tyler responded that:

* The core team’s intent is **not to build a commercial audit business**.
* The focus remains **marketplace trust and provider reliability**.
* Pricing strategy has not yet been finalized.
