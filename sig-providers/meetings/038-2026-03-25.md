## Meeting Notes

### Opening Remarks & SIG Providers Overview

* Tyler Wright opened the session and welcomed participants to the SIG Providers meeting
* Explained that SIG Providers focuses on all aspects of provider operations and infrastructure
* Highlighted recent network and provider upgrades as key discussion areas
* Encouraged participants to raise additional topics during the meeting
* Noted availability of provider-specific announcements in Discord for real-time updates

### Network Upgrade — Mainnet 17

* Recent upgrade to Akash Mainnet 17 completed, introducing Burn Mint Equilibrium and smart contracts
* Providers required to upgrade to remain compatible with network changes
* Significant community and core team testing conducted prior to deployment
* Estimated that approximately half or more of providers had completed upgrades at time of meeting
* Emphasis on ensuring all providers upgrade to maintain participation in workload bidding
* Reminder that outdated providers may not be able to bid on workloads

### Provider Updates & Bug Fixes

* Provider upgrade introduced support for AKT-based changes related to BME
* Fix implemented for stale open bids issue
* Previous bug caused AKT to be locked in bid escrow, limiting provider liquidity
* Bug successfully resolved and deployed with BME update
* Additional edge cases related to bid locking currently being addressed
* Multiple pull requests in testing to further improve reliability

### Gateway API Migration

* Migration underway from Kubernetes ingress controller to Gateway API
* Change driven by deprecation of current ingress implementation
* Gateway API provides improved long-term support and scalability
* Extensive testing conducted with positive results
* Final validation in progress following BME upgrade
* Expected release of migration update within the following week

### Provider Stability Improvements

* Ongoing work to address provider crashes and runtime panics
* Multiple bug fixes currently in development and testing
* Focus on improving overall reliability and uptime
* Continuous improvements being rolled out incrementally

### Provider Console Updates

* Provider Console updated to support Mainnet 17 and BME
* Users can now verify provider version directly within settings
* Upgrade process requires updating node first, followed by provider software
* Improvements planned for wallet interface and AKT display
* Future updates will include design enhancements and usability improvements

### Akash Home Node Progress

* Home Node undergoing active testing and development
* Focus on GPU-based workloads and inference capabilities
* Testing includes performance, latency, and remote execution scenarios
* Home Node users act as worker nodes within the network
* Beta access currently limited to users with supported GPUs such as 4090 and 5090
* Additional GPU types being considered for future participation
* Encouragement for users to sign up for early access and testing

### Provider Onboarding & CLI Migration

* Discussion raised about migrating from CLI-based provider setup to Provider Console
* Confirmed that migration is supported and follows similar workflow
* Users can connect wallet and manage provider through console interface
* Console simplifies upgrade and management processes
* Documentation for migration process currently in development
* Support available via Discord for users encountering issues

### Community Support & Collaboration

* Encouragement for experienced providers to assist others in community channels
* Emphasis on maintaining up-to-date provider versions across the network
* Recognition of active contributors supporting others in Discord
* Highlighted importance of sharing knowledge and troubleshooting collaboratively

### Documentation & Communication

* Ongoing communication through provider announcements channel in Discord
* Core team actively sharing upgrade instructions and best practices
* Continued effort to improve clarity and accessibility of provider resources

### Upcoming Provider Audits

### Closing Remarks
