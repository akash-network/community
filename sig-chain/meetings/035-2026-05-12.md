### Oracle Performance Improvements

* Artur discussed improvements being made to the Oracle subsystem.
* The objective is to improve:

  * Query performance.
  * Data management efficiency.
  * Long-term scalability.
* The current Oracle implementation stores historical information that is no longer required for operational purposes.
* Planned improvements include:

  * Pruning unnecessary historical data.
  * Optimizing Oracle queries.
  * Improving indexing and retrieval efficiency.
  * Reducing storage overhead.
* Artur confirmed that this work has already been completed and fully tested.
* The improvements are expected to increase overall performance while simplifying Oracle maintenance requirements.

### Resource Reclamation Feature

* Artur introduced the resource reclamation feature that is scheduled for inclusion in the upcoming network upgrade.
* The feature is designed to improve coordination between providers and tenants.
* Under the new mechanism:

  * Providers can identify resources that need to be reclaimed.
  * Providers can submit a formal reclamation request.
  * Tenants receive advance notification before resources are reclaimed.
* The system creates a grace period during which workloads can prepare for migration or transition.
* Providers will no longer be required to abruptly terminate leases when reclaiming infrastructure resources.
* Benefits discussed include:

  * Better user experience.
  * Improved workload stability.
  * Reduced disruption for tenants.
  * More predictable infrastructure management.
* Artur emphasized that the blockchain portion of the feature has already been completed.
* Final validation is currently focused on provider-side implementation and integration.

### Oracle State Management Changes

* Additional discussion covered Oracle state handling.
* Artur explained that Oracle state will be rebuilt rather than migrated.
* Historical Oracle information is already preserved through blockchain event data.
* Rebuilding Oracle state simplifies maintenance and reduces unnecessary complexity.
* Because historical records remain accessible through blockchain events:

  * No critical information is lost.
  * State management becomes more efficient.
  * Operational overhead is reduced.
