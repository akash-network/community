## Meeting Notes

### Opening Remarks & SIG-Chain Scope

* Tyler Wright welcomed participants to the January SIG-Chain monthly meeting
* He clarified that SIG-Chain focuses on Akash blockchain protocol topics
* Areas of focus include validator relations, network upgrades, and core chain functionality
* Tyler distinguished SIG-Chain from provider and node-focused engineering syncs
* Context was provided that the previous SIG-Chain meeting took place in November

### Context From Previous Updates

* Tyler referenced the prior update from Artur covering the Cosmos SDK v53 migration post-mortem
* Smart contracts and the upcoming BME were identified as the next major milestones
* December did not include a SIG-Chain meeting due to holidays and engineering focus

### Engineering Update ‚Äì Smart Contracts, Oracle, and BME

* Artur Troian provided a comprehensive update on core protocol progress
* He confirmed that development work on smart contracts, Oracle, and BME was completed before the new year
* These features require careful implementation due to pricing correctness and network impact
* The Oracle module is critical for accurate price feeds and cannot be rushed
* The team reviewed other ecosystem implementations to avoid known pitfalls

### Testing Status and Code Readiness

* Internal testing of smart contracts and price feeder functionality has been conducted
* Oracle pricing was confirmed to be functioning correctly in internal tests
* The Oracle module is considered feature complete, with further bugs potentially identified during testing
* BME is also feature complete from a functionality standpoint
* Artur noted that BME required significant redesign of core blockchain ‚Äúplumbing‚Äù
* Changes impacted deployments, the marketplace, and APIs
* The scope of these changes extended development timelines beyond initial estimates
* All related code was published and entered the release process as of the prior week

### Testnet Phases and Community Participation

* An internal testnet is currently being set up by the core team
* The goal of phase one is to identify critical bugs before public exposure
* Scott Carruthers is actively working on integrating Oracle pricing for BME testing
* Once phase one completes, the team will move to phase two
* Phase two will be a public, incentivized testnet open to the community
* Final participant requirements are still being defined
* Details and instructions will be published once finalized
* Tyler Wright will act as the primary coordination point for the testnet

### Network Upgrade Strategy

* Artur explained that Oracle and BME will be released in two separate network upgrades
* The first upgrade will introduce the Oracle module only
* This allows the network to validate Oracle stability and price accuracy on mainnet
* Once confidence is established, the BME upgrade will be triggered
* The BME upgrade is more complex due to changes in escrow and marketplace data structures

### BME Design and Fallback Mechanism

* BME introduces support for multiple pricing mechanisms
* A fallback mechanism is implemented to ensure uninterrupted deployments
* Deployments will include both AKT-per-block and USDC-per-block pricing
* If the BME circuit breaker triggers, deployments continue using AKT pricing
* Once resolved, payments automatically transition to USDC
* This design ensures continuity even during market or oracle disruptions

### Governance and Smart Contract Deployment

* Jannick H asked whether smart contracts would be permissioned
* Artur confirmed that the current model requires governance proposals for contract upload
* Only the governance module is whitelisted for contract deployment and instantiation
* The possibility of whitelisting additional addresses was discussed
* Multisig-based whitelisting could allow faster upgrades if approved via governance
* The team acknowledged trade-offs between flexibility and governance control
* Further discussion is expected to continue on GitHub

### Announcements, Validator Communication, and Closing

* Tyler confirmed that validators will be notified via Discord and email lists
* Announcements will cover testnet phases and upcoming network upgrades
* Community members were encouraged to help stress-test BME before mainnet deployment
* Tyler thanked Artur and the core engineering team for their work
* Participants exchanged New Year greetings
* The meeting concluded with appreciation and reminders of upcoming SIGs and working groups
