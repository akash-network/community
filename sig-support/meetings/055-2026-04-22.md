## Meeting Notes

### Opening Remarks & SIG Support Overview

* Tyler Wright opened the meeting and welcomed participants to the monthly Support Special Interest Group session
* Explained that SIG Support focuses on all open support-related issues across the Akash ecosystem
* Topics highlighted included provider issues, blockchain node issues, support workflows, documentation, and user experience improvements
* Tyler Wright explained that support issues are actively managed through the support repository inside the Akash Network GitHub organization
* Community members encouraged to use Discord and GitHub issues for support discussions and reproducible issue reporting
* Tyler Wright emphasized the importance of including logs and detailed technical information when submitting support issues
* Mentioned that console-related issues are managed separately through the Console repository issue tracker
* Tyler Wright acknowledged the ongoing efforts of the Akash Core Engineering team and community contributors assisting with support and issue triage

### Support Repository Triage Process

* Scott Carruthers explained the standard issue triage workflow followed during SIG Support meetings
* Purpose of the process described as ensuring all issues are assigned to the correct repositories and engineering owners
* Scott Carruthers reported that there were currently no untriaged issues in the support repository
* Explained that recent issues had already been reviewed and assigned appropriately shortly after submission
* Mentioned that the absence of pending triage does not mean there have been no new issues
* Community contributors acknowledged for actively assisting with issue management and support workflows

### Burn Mint Equilibrium & Oracle v2 Updates

* Scott Carruthers provided updates regarding post-upgrade work following the Burn Mint Equilibrium network upgrade
* Core Engineering team identified optimization opportunities within the pricing oracle implementation
* Oracle Version 2 currently under extensive testing and validation
* Scott Carruthers emphasized that Burn Mint Equilibrium is a highly sensitive financial mechanism requiring extremely careful testing
* Testing currently includes weighted average pricing calculations, deviation handling between pricing sources, and circuit breaker logic
* Oracle v2 testing also includes warning and hold state validation scenarios
* Scott Carruthers stated that Oracle v2 development is effectively in code freeze and approaching release readiness
* Upcoming network upgrade expected to introduce Oracle v2 once testing is fully completed
* Tyler Wright and Scott Carruthers both emphasized the importance of stability and reliability before deployment

### Lease Notification Window Feature

* Scott Carruthers introduced a new lease notification feature being developed for Akash providers
* Feature designed to provide deployers with advance warning before workloads are terminated by providers
* Users will be able to specify notification windows during lease creation
* Example provided where deployers may request a two-hour warning before workload shutdown
* Providers will not be able to terminate workloads before the agreed notification window expires
* Scott Carruthers described the feature as relatively straightforward and expected to be completed quickly
* Planned for inclusion alongside Oracle v2 in the next network upgrade

### Confidential Computing & Trusted Execution Environments

* Scott Carruthers discussed ongoing confidential computing and trusted execution environment development
* Confidential compute described as highly important for AI developers requiring secure data protection during model training
* Core Engineering team actively accelerating confidential computing development efforts
* Work includes testing and evaluation of secure enclave technologies
* Tyler Wright later confirmed that confidential computing remains a major roadmap priority for enterprise and AI adoption

### Gateway API Migration & Provider Updates

* Scott Carruthers explained that the Core Engineering team is preparing providers for migration away from deprecated Kubernetes ingress controllers
* Existing ingress controller implementations no longer receiving security patches from Kubernetes maintainers
* Akash providers therefore required to migrate toward Gateway API architecture
* Migration already code complete and release prepared
* Additional migration details scheduled for discussion during the upcoming SIG Providers meeting
* Tyler Wright emphasized that the migration has major implications for provider infrastructure and long-term security

### Akash CLI Redesign

* Scott Carruthers revealed that a major redesign of the Akash CLI is currently underway
* New CLI expected to introduce streamlined context management for switching between networks such as sandbox, testnet, and mainnet
* Current long-form command structures expected to be simplified significantly
* New CLI also planned to support improved governance, provider, lease, and network visualizations directly inside terminal workflows
* TEE-related functionality planned for integration into the new CLI
* Scott Carruthers explained that the redesign aims to improve usability and developer experience significantly
* Early implementation work already progressing internally

### Community Documentation & Open Source Contributions

* Tyler Wright emphasized that all Akash repositories remain fully open source and community contributions are encouraged
* Community members encouraged to submit feature requests, documentation improvements, and support-related feedback
* Tyler Wright acknowledged that the Core Engineering team cannot solve every issue immediately and community participation remains extremely valuable
* Documentation contributions highlighted as an excellent entry point for new contributors
* Tyler Wright encouraged developers to set up local environments and participate in ecosystem development whenever possible

### Community Support Themes & User Experience

* Tyler Wright invited Rodri R to share any recurring support themes from Discord and community support channels
* Rodri R stated that most current support requests remain relatively standard
* Common support topics currently include provider setup questions and deployment troubleshooting
* Rodri R also mentioned the upcoming release of the support bot inside Discord support workflows

### Akash Support Bot Launch

* Joe Deng provided updates regarding the Akash Support Bot currently being tested internally
* Support Bot currently active inside the hackathon support section within Discord
* Vanguard members actively assisting with live testing and supervision of bot responses
* Bot designed to answer common documentation and onboarding questions automatically
* Joe Deng explained that hackathon participants will use the support bot during upcoming events
* Human Vanguard support staff will remain available to supplement and correct responses when necessary
* Feedback gathered during hackathons expected to improve the bot’s training and long-term performance
* Joe Deng explained that users can interact with the bot using the `@akash support` command
* Bot currently performs best on documentation-related and standard onboarding questions
* Long-term goal is continuous learning and iterative improvement using real user interactions

### Provider Verification & Audit Improvements

* Tyler Wright announced that the Core Engineering team is preparing specifications for improved provider verification and audit workflows
* Current provider verification process described as highly manual
* Future implementation expected to automate portions of provider auditing and validation
* Public specification draft planned for community review and feedback before implementation begins
* Tyler Wright explained that improving provider verification is important for deployer confidence and network trustworthiness

### Closing Remarks

* Encouraged continued participation through Discord, GitHub issues, and documentation contributions
* Reminded attendees about the upcoming SIG Providers meeting immediately following the session
