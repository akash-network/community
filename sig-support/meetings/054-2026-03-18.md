## Meeting Notes

### Opening Remarks & SIG Support Overview

* Tyler Wright opened the session and welcomed participants to the SIG Support meeting
* Explained that SIG Support focuses on issues related to node, provider, and blockchain support
* Clarified that the support repo is the primary location for core issues
* Noted that console-related issues should be handled in a separate repository
* Encouraged participants to propose additional agenda items during the session
* Highlighted growing interest in support automation including support bots

### Issue Management — Repo Clarification

* Identified confusion within the community regarding where to open issues
* Several issues incorrectly opened in the support repo were actually console-related
* Examples included wallet API issues, credit card issues, and console-specific bugs
* Decision made to migrate these issues to the appropriate console repository
* Emphasis on guiding community to use correct repositories going forward
* Acknowledged that confusion exists even among experienced contributors

### Support Repo — Issues Awaiting Triage

* Remaining issues after filtering console-related ones were primarily provider-related
* First issue involved incorrect GPU availability reporting after lease closure
* Inventory operator showing zero GPUs instead of expected availability
* Issue identified as provider codebase bug
* Assigned to relevant core team members for further investigation

### Home Node — Geo Location Issue

* Issue raised regarding incorrect geo-location attributes for providers
* Example where node labeled as North America was actually located in Croatia
* Identified as potential mismatch between metadata and actual node location
* Sandeep provided context regarding latency issues caused by incorrect location reporting
* Core team acknowledged issue and committed to auditing provider attributes
* Assigned to Zeke for further investigation and resolution

### Provider Crashes — Issue Review

* Two issues identified related to provider crashes under different conditions
* One issue involved panic after startup
* Another involved crash during broadcast message handling
* One issue already had an active pull request and fix awaiting review
* Second issue determined to be caused by use of development chart not intended for mainnet
* Recommendation to close invalid issue caused by unsupported configuration

### Issue Triage Process

* Core team demonstrated structured approach to triaging issues
* Issues assigned to relevant team members immediately during call
* Clear categorization into provider, console, or other domains
* Emphasis on reducing backlog in “awaiting triage” state
* Encouraged community to provide detailed context when reporting issues

### Community Support Insights

* Tyler invited feedback from active community members on support trends
* Rodri raised observation regarding BME testnet visibility in console
* Clarification provided that visible BME testnet is intentional for internal testing
* Not part of public testnet and not intended for community usage
* Previous incentivized testnet environment has already been decommissioned
* New testnet instance created specifically for internal development

### BME Testing & Support Readiness

* Continued internal testing of BME within console environment
* Testing focused on ensuring readiness for Mainnet 17 upgrade
* Recognition of extensive prior testing via incentivized testnet
* Over 200 participants contributed to testing efforts
* Approximately 74 participants received rewards
* Strong validation of system stability ahead of deployment

### Documentation & Upgrade Preparation

* Documentation for upcoming Mainnet 17 upgrade already shared
* Validators, exchanges, and ecosystem participants being prepared
* Emphasis on clear communication to minimize upgrade issues
* Recognition of improved observability and tooling from past upgrades

### Closing Remarks

* Tyler Wright thanked participants and core team members for contributions
* Encouraged community members to continue engaging through SIG Support
* Highlighted opportunity to contribute through issue triage, spec writing, and fixes
* Reinforced importance of proper issue reporting and repository usage
* Meeting concluded with invitation for continued discussion in Discord
